在github上搭建博客

1、在Github上建立相应的仓库

既然想在GitHub上建立博客，前提是有GitHub账号的，没有的就注册一下。

新建仓库，直接点击右上角“+”号菜单，其中就有New repository这个子菜单，接着就是填写仓库对应资料的问题了。

Repository name这个选项一定要填你的username.github.io不然会出现各种问题（参考资料这么说的）还有就是下方的初始化Initialize this repository with a README勾选，这样能将Github中这个仓库上传的代码clone到你的电脑中。

之后点击创建就可以了。

那么怎么知道自己的博客建立了呢，点击右上角公文包的菜单，选择子菜单Your repositories

进入你刚创建的仓库，会有个Settings的按钮，点击后下拉到GitHub Pages，后面有个默认的地址，它就是你的博客地址了。想要一个特别的域名的话，可以买一个域名，在setting里面绑定就可以了（后面的资料链接有对应的操作）

完成以上操作，仓库就建好了，接着就是配置本机环境。

2、搭建环境

安装node.js和git.默认安装即可。未安装的按一下链接参考

https://www.cnblogs.com/zhouyu2017/p/6485265.html

安装完成后，cmd打开命令窗口分别输入node -v, npm -v, git –version, 都返回版本号就说明安装好了。

ssh授权，打开git bash 输入

ssh key检查是否存在：

$ ls ~/.ssh/id_rsa.pub如果出现文件路径则存在

生成ssh key：

$ ssh-keygen -t rsa -C “your@email.com“

-t 指定密钥类型，默认是 rsa ，可以省略。

-C 设置注释文字，比如邮箱。

-f 指定密钥文件存储文件名。

以上代码省略了 -f 参数，因此，运行上面那条命令后会让你输入一个文件名，用于保存刚才生成的 SSH key 代码。连续3个回车。如果不需要密码的话。 最后得到了两个文件：id_rsa和id_rsa.pub。

如果你是第一次使用git，在此步骤前先设置自己的用户名和邮箱

添加密钥到github上：

打开你的id_rsa.pub文件，拷贝内容粘贴到github上的ssh设置里就可以了。

复制，ssh-rsa…到最后。

具体操作：登录你的github账号，点击右上角设置，点击Settings->点击左边菜单栏的 SSH key，添加一个新的sshkey,将代码贴入，授权完成。

测试：

$ ssh -T git@github.com

第一次的话期间若出现是否continue，输入yes，出现hi。。。（加上你的用户名）就成功了。

3、安装Hexo

新建一个文件夹作为你博客文件存储的地方，打开git bash进入这个文件夹，输入

npm install hexo-cli -g

开始安装Hexo,之后输入

hexo -v检查hexo是否安装成功。

然后输入hexo init初始化该文件夹。

最后的最后尝试：输入hexo s

地址栏中输入http://localhost:400/可以看到一个界面，由于我的主题已经更改，所以copy一个别人的图给大家看看是什么样子。

这个是默认的主题，如果不满意可以更改，这个在后面会整理出对应的资料。有兴趣的小伙伴可以关注一下我的博客。

4、写文章

输入hexo new post ‘filename’

之后再source->_posts中你可以看到对应的.md文件，是的，这就是文章保存的位置，点击各个文件，就可以开始编辑。

5、上传项目

打开_config.yml，修改repository后面的内容(在github对应仓库里的ssh复制过来)，注意：type、repository、branch冒号的后面都有一个空格

之后输入npm install hexo-deployer-git -save

之后编辑好文章，执行命令：hexo d -g（生成和部署）然后就可以访问你的域名了，可以看到你刚编辑的文章。

mac:

sudo ssh -T git@github.com #报permission denied (publickey)

sudo ssh -T -v git@github.com #检查进程，发现

发现它一直在查找id_rsa、id_dsa、id_ecdsa等文件，然后就没了

解决：

ssh-add -l #显示私钥列表

The agent has no identities.

如果是这样，就是没有添加成功私人密钥

ssh-add ~/.ssh/id_rsa #这个id_rsa应是你命名的私人密钥默认名，在尝试ssh-add -l就发现ok了。
